## 1. ゲームの概要

- 2人で対戦するカードゲーム。
- 自分の「キャラクター」（枠3つぶん）と、50枚の「デッキ」を使って戦う。
- スキルカードで相手のキャラクターにダメージを与えていく。
- 対戦相手は、人（PvP）またはコンピューター（PvE）。
  ※プロトタイプではコンピューター戦だけ動いていた。

### 勝ち負け（2026-07-22 社長決定）

次のどちらかを満たしたら勝ち。

1. **相手のキャラクターを全て戦闘不能にする**
2. **相手の山札が0枚になり、相手がドローフェーズで手札を5枚にできない**

（逆に、自分がどちらかの状態になったら負け）

## 2. カードの種類

カードは全部で 4 種類ある。

| 種類 | 英語名 | 役割 |
|---|---|---|
| キャラクターカード | character | 場に出て戦う本体。HP と属性を持つ |
| スキルカード | skill | 攻撃・防御・回復などの技 |
| 装備カード | equipment | キャラクターに付けて属性などを足す |
| フィールドカード | field | 場全体のルールを変える |

### 全カードに共通する情報

- **id**: カードごとの番号（例: `1-A001-LSR`。「弾-通し番号-レアリティ」の形）
- **vol**: 収録弾（第1弾なら 1）
- **name**: カード名
- **rarity**: レアリティ。低い順に **C → UC → R → SR → SSR → USR → LSR** の7段階
- **effectText**: 効果の文章
- **flavorText**: 世界観の文章（ルールには関係ない）

### キャラクターカードだけの情報

- **size**: 大きさ。`normal`（普通）と `legendaryLarge`（伝説級の大型）の2種類
  - **大型はキャラクター枠を2つ使う**（2026-07-22 社長決定。→ 4章）
- **hp**: 体力。第1弾では 5〜20
- **attribute**: 属性（0〜5個持つ。0個は「[魔神素体]グロウ」だけで効果で増やしていくデザイン。5個は「[朽ち往く魔王]トランザード」だけ）

### スキルカードだけの情報

- **costAp**: 使うのに必要な AP の数（第1弾では 0〜8）
- **conditionAttribute**: 使うための属性の条件（下の「6. スキルを使う条件」を見る）
- **baseValue**: 効果の基本の数値（ダメージ量など。第1弾では 0〜9）
- **valueType**: 技の種類。4つある
  - `attack` = 攻撃
  - `guard` = 防御。**相手の攻撃に割り込んで使い、その攻撃1回のダメージを baseValue 分減らす**。
    自分のターンには使えない（2026-07-22 社長決定。くわしくは6章の「割り込み」）
  - `support` = 補助。効果文どおりの働き（共通ルールで決めることが残っていれば【要決定】）
  - `heal` = 回復

### 装備カードだけの情報

- **addAttribute**: 付けたキャラクターに追加される属性
- ルールは5章の「装備カードのルール」を見る（2026-07-22 社長決定）

### フィールドカードだけの情報

- 追加の数値情報はなく、効果の文章だけを持つ
- ルールは5章の「フィールドカードのルール」を見る（2026-07-22 社長決定）

## 3. 属性

第1弾のカードには、全部で 17 種類の属性が出てくる。

- 武器系: 斬（ざん）・突（とつ）・打（だ）・射（しゃ）・飛（ひ）
- 元素系: 炎・氷・雷・風・土・木・聖・闇
- 種族・役割系: 竜・獣・補・守

属性は次の 2 つの意味で使う。

1. **キャラクターが持っているもの**（例: クラウディアは「斬・聖・雷」）
2. **スキルを使うための条件**（例: 条件「斬・斬」のスキルは、斬を2個以上持つキャラクターしか使えない）

装備カードやフィールドカードで属性を後から足すこともある。

## 4. デッキのルール

- デッキは次の 2 つで 1 セット。
  - **キャラクター: 枠3つぶん**（普通のキャラ=1枠、大型 legendaryLarge=2枠。
    例: 普通×3、または 大型×1＋普通×1）（2026-07-22 社長決定）
  - **カード（スキル・装備・フィールドなど）: 50枚**（2026-07-22 社長決定。最初は40枚だったが、
    山札切れ決着に偏りすぎたためシミュレーションで検証して50枚に変更）
- 同じカードは **4枚まで** 入れられる（2026-07-22 社長決定。同上の理由で3枚から変更）
- キャラクターとして選んだカードをデッキの50枚にも入れられるかは【要決定】
  （サンプルデッキでは両方に入っていた）

## 5. バトルの場とターンの流れ

### バトルの場

それぞれのプレイヤーが持つ場所:

- **デッキ（山札）**: 裏向きの50枚。ここからカードを引く
- **手札**: 引いたカード
- **APエリア**: 手札から置いた「エネルギー」のカード置き場
- **キャラクター場**: キャラクター（枠3つぶん）が並ぶ場所
- **トラッシュ**: 使い終わったカードの置き場

2人で共有する場所:

- **フィールドカード置き場**（1枚だけ）

### バトルの準備（2026-07-22 社長決定）

1. 先攻・後攻を **ランダム** に決める
2. お互いデッキ50枚をシャッフルする
3. お互い手札を **5枚** 引く
4. キャラクター（枠3つぶん）を場に出す（この並び順が、アクターの交代の順番になる）
5. **後攻のプレイヤーは、デッキの上から2枚を APエリア にチャージした状態で始める**（後攻の補償）
6. 先攻のプレイヤーからターンを始める

### ターンの流れ（2026-07-22 社長決定）

自分のターンは次の順番で進む。

1. **ドローフェーズ**: 手札が **5枚になるまで** 山札から引く
   - すでに5枚以上あるなら引かない
   - 山札が0枚で手札を5枚にできなかったら、その時点で負け（→ 1章の勝ち負け）
2. **プレイフェーズ**: 手札からカードを使える
   - スキルカードを使えるのは、今の **アクター**（条件は6章）
   - **スキルを使うたびに、アクターは次のキャラクターに交代する**（→ 6章）
   - AP がある限り、スキルは **1ターンに何回でも** 使える
   - キャラクターカードも使える（→ このすぐ下）
3. **チャージフェーズ**: 手札から **好きな枚数** のカードを選んで、APエリアにチャージできる
   （置いたカードはエネルギー扱いになり、手札には戻らない）
4. **ターンエンド**: 相手のターンになる

### キャラクターカードを使った時の効果（2026-07-22 社長決定）

- 手札の **キャラクターカード** を使うと、場にいる **同じ名前の自分のキャラクター** のダメージを **2回復** する。
- 使ったキャラクターカードはトラッシュに送られる。
- 場に同じ名前のキャラクターがいない時は **使えない**（AP のチャージには使える）。
- キャラクターカードを使っても、アクターは交代しない（交代するのはスキルを使った時だけ）。

### 装備カードのルール（2026-07-22 社長決定）

- プレイフェーズに、手札から **コストなし** で使える。
- **生きている味方1枚** に付ける。1キャラに付けられる装備は **1つだけ**。
- すでに装備しているキャラに新しい装備を付けてもよい（**付け替え**）。古い装備はトラッシュへ。
- キャラが戦闘不能になっても装備は **付いたまま**（復活したらまた効果が戻る）。
- 装備カードの addAttribute は、付いている間そのキャラの属性として数える。
- 装備カードを使っても、アクターは交代しない。

### フィールドカードのルール（2026-07-22 社長決定）

- プレイフェーズに、手札からコストなしで場に出せる。
- 場に置けるフィールドは **1枚だけ**（両プレイヤー共有）。
- 新しいフィールドを出すと **上書き** される。古いフィールドは元の持ち主のトラッシュへ。
- フィールドの効果は、両プレイヤーの全キャラクターに働く。
- フィールドカードを使っても、アクターは交代しない。

## 6. AP とスキルを使う条件

### AP（エネルギー）

- 手札のカードを APエリア に置くと、1枚 = 1AP になる。
- どのカードでも AP にできる。
- スキルを使うと、そのコストの数だけ APエリア のカードが **トラッシュに送られる**（使い捨て。毎ターン回復はしない）。
- 使ったスキルカード自体もトラッシュに送られる。

### スキルを使う条件（2つとも必要）

1. **AP が足りている**: 自分の APエリア の枚数 ≧ スキルの costAp
2. **属性の条件が合っている**: 自分の **アクター** の属性が、スキルの conditionAttribute を全部含んでいる
   - 同じ属性が2個必要なら、キャラクターも2個以上持っていないとダメ（例: 条件「竜・竜」→ 竜を2個以上）
   - 装備・フィールド・効果で追加された属性も数えてよい

### アクター（2026-07-22 社長決定）

場のキャラクターのうち「今、前に出て戦っている1枚」を **アクター** と呼ぶ。

- バトル開始時のアクターは、並び順の **1番目** のキャラクター。
- スキルを使えるのは **アクターだけ**。属性の条件も、アクターの属性で見る。
- **自分のターンにスキルを使うと、アクターは並び順で次のキャラクターに交代する**（1 → 2 → 3 → 1 と順番に回る）。交代先を選ぶことはできない。
- **相手のターンにスキルを使った時（割り込み）は、アクターは交代しない。**
- 攻撃スキルが狙えるのは **敵のアクターだけ**。「敵全体」などとカードに書いてある時だけ例外。
- 戦闘不能のキャラクターは、交代の順番から飛ばされる。
- アクター自身が戦闘不能になったら、並び順で次の生きているキャラクターに **強制交代** する。

この仕組みのポイント: スキルを連発するとアクターがどんどん交代していくので、
「誰が・どの順番で・どの属性のスキルを使うか」がデッキ作りと戦略の中心になる。

### 割り込み（guard）（2026-07-22 社長決定）

- 相手が攻撃スキルを使った時、攻撃される側は **ダメージを受ける前に** 手札から guardスキルを使って割り込める。
- 割り込みで使えるのは **guardスキルだけ**。使えるタイミングは **攻撃された時だけ**。
- AP コストと属性条件は普通に必要（属性は自分のアクターで判定）。
- その攻撃1回のダメージを baseValue 分減らす。複数枚重ねて使ってもよい（軽減は合算）。
- 割り込みではアクターは交代しない。
- 割り込みのポイント: AP を攻撃に使い切るか、相手のターンに備えて守りに残すか、の駆け引きになる。

## 7. ダメージ・回復・戦闘不能

- ダメージはキャラクターごとに **ダメージカウント** として貯まっていく。
- 残り HP = 「カードの HP」−「ダメージカウント」
- 回復はダメージカウントを減らす（0 より下にはならない = HP は最大値を超えない）。
- ダメージカウントが HP 以上になったら、そのキャラクターは **戦闘不能**（プロトタイプではカードが裏返る表現だった）。
- 戦闘不能になったキャラクターは **場に残る**。「復活回復」のカードで戦えるように戻せる。
- 戦闘不能のキャラクターはアクターの交代順から飛ばされる。アクター自身が戦闘不能になった時は、次の生きているキャラクターに強制交代する（→ 6章）。
- guard（防御）スキルは、相手の攻撃への割り込み専用（→ 6章の「割り込み」）。

## 8. カードの効果文に出てくる言葉（用語集）

第1弾のカード144枚の効果文から読み取れた言葉。正式な定義は【要決定】のものが多い。

| 言葉 | 意味（読み取れる範囲で） |
|---|---|
| アクター | 前に出て戦っているキャラクター（→ 6章） |
| 戦闘不能 | HP が 0 になった状態（→ 7章） |
| チャージ | カードを APエリア に置くこと |
| トラッシュ | 使い終わったカードの置き場。「デッキからトラッシュ」等の効果もある |
| ドローフェーズ | ターンの最初にカードを引く時間 |
| 復活回復 | 戦闘不能のキャラクターを回復して戦えるように戻すこと |
| 敵全体 | 相手のキャラクター全部 |
| ○属性×1ダメージ | 対象が持つその属性の数だけダメージを足す、の意味らしい |
