# カードマスター管理（管理画面と弾の運用）

第1弾のカードプールが完成し、第2弾以降を作っていくための仕組み。
他社TCG（MTG/Scryfall・ポケカ・遊戯王）のデータ管理を調べて設計した。

## 基本の考え方

- **カードは「弾（セット）」に属する**。弾のメタ情報は `data/sets.json`。
  社長の言う「弾数」と「テーマNo.」を別フィールドで持つ（MTG の set と block の二軸に対応）。
- **公開されるのは `status: 'released'` の弾のカードだけ**。制作中の弾は `status: 'draft'`。
- **制作中カードは公開ビルドにもGitにも入らない**。`data/wip/`（gitignore）に置く。
  静的配信では「クライアントに入ったデータ＝誰でも読める」ため、UIを隠すのではなく
  **データ自体を公開ビルドから外す**のが唯一確実な方法（Hearthstone等の実際のリーク事例に基づく）。

## 管理画面の起動

```
npm run admin
```

ローカル専用（`http://localhost:5273`）。**公開サイトには一切デプロイされない**ので、
そもそもネット上に存在せず最も安全。`data/` のJSONを直接読み書きする。

### できること

- 弾タブで弾を切り替え／新しい弾を作る
- 弾のメタ情報（弾数・テーマNo.・テーマ名・サブタイトル・パックタイプ・状態）を編集
- カードの一覧・検索・フィルタ（種類/レア/未実装のみ）
- カードの追加（弾内で `A001` から自動連番）・編集・削除
- **効果の実装状況を可視化**（効果文があるのに `engine/src/effects/` に実装が無いカードを赤表示）
- **公開前チェック**（画像の欠け・基本値の理論値ずれ・ID重複を赤緑で判定）
- スキルの基本値の理論値（`(コスト+1)×2−1＋(条件属性数−1)＋盾ガード+1`）を表示・ワンタップ適用

### 保存先の自動振り分け

- 公開弾の公開カード → `data/cards.json`（Gitで管理・公開される）
- 制作中カード（弾が draft、またはカード個別が draft） → `data/wip/cards.volN.json`（**gitignore・非公開**）

## 新しい弾（例: 第2弾）を作る流れ

1. `npm run admin` を起動
2. 「＋ 新しい弾」→ メタ情報を入力。**状態は draft のまま**。
   テーマ名は公開直前まで「コードネーム」で伏せてもよい（コミット履歴からの漏れ防止）
3. 「＋ カード追加」でカードを作る。draft弾のカードは `data/wip/` に保存され、公開されない
4. 効果は `engine/src/effects/vol2.ts`（新規）に書き、`effects/index.ts` へのマージは
   **リリース時だけ**にする（import した瞬間に公開JSへ焼き込まれるため）
5. 「公開前チェック」が全て緑になったら弾の状態を `released` にし、
   `data/wip/cards.volN.json` のカードを `data/cards.json` へ移す（管理画面が振り分ける）
6. push すると CI が `npm run check:leak` で「未公開弾の情報が公開ビルドに混入していないか」を
   最終チェックし、問題なければ公開される

## 安全網（多層防御）

1. **engine のゲート**: `ALL_CARDS` は released の弾のカードだけ（`engine/src/cards.ts`）
2. **物理分離**: 制作中カードは gitignore された `data/wip/` にしか無い
3. **CI の最終スキャン**: `scripts/check-no-wip-leak.mjs` が公開ビルドJSを走査し、
   未公開のテーマ名・コードネーム・制作中カード名が混入していたらデプロイを止める
4. **テスト**: `engine/test/sets.test.ts` が「公開カードは全て released 弾」「画像の欠け・stray画像」を検査

## データ形式

### data/sets.json

```json
{
  "sets": [
    {
      "vol": 1,
      "themeNo": 1,
      "themeName": "聖戦残火",
      "themeSubtitle": "禍いの足音",
      "packType": "DX",
      "status": "released",
      "releasedAt": "2026-07-01",
      "codename": ""
    }
  ]
}
```

### カード（data/cards.json / data/wip/cards.volN.json）

既存の `1-A001-LSR`（`{vol}-{code}-{rarity}`）形式のまま。1枚も変更していない。
`status?: 'draft' | 'released'` を任意で追加できる（省略＝弾の状態に従う）。

## 将来の再録・エラッタに備えて

今は各カードの id にレアリティが焼き込まれている（`1-A001-LSR`）。
第2弾以降で「同じキャラを別レアで再録」が始まったら、Scryfall/遊戯王方式の
「実体ID（oracleId）＋印刷（printing）」の二層モデルへ拡張する余地を残してある
（`oracleId` を任意フィールドとして足すだけで移行できる）。今は不要なので入れていない。
