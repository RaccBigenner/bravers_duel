# デッキ枚数の変更: 40枚/3枚 → 50枚/4枚（2026-07-22 社長決定）

## 背景

山札切れ決着が全体の34%を占め、しかも「特定カードの自己トラッシュ（ジエンド・ミルオン）」と
「雷土の重撃の耐久戦法」に偏って発生していた。狙いの「倒して勝つゲーム」に寄せるため、
デッキ枚数の変更をシミュレーションで検証した。

## A/Bテスト（アーキタイプ8種総当たり、各1,400戦、simpleAI）

| 条件 | 山札切れ決着 | 全滅決着 | 平均ターン | 先攻勝率 |
|---|---|---|---|---|
| 40枚・同名3枚（旧） | 34.1% | 65.9% | 16.0 | 48.9% |
| 50枚・同名3枚 | 6.4% | 93.6% | 16.2 | 48.3% |
| **50枚・同名4枚（採用）** | **4.7%** | **95.3%** | 15.6 | 50.6% |

- 効いているのは主に50枚化。同名4枚は安定感の上乗せ
- 雷土の「相手を山札切れに追い込む」戦法はほぼ消滅（山切勝ち155→36回）
- ジエンドの山札切れ負けは151→29回に減少（それでも自己トラッシュのリスクは残る）

## 決定

**デッキ50枚・同名4枚まで** を公式ルールにする（GAME_RULES v0.7）。
実験用の切り替えは `npm run sim -- --deckSize N --maxCopies N` で今後も使える。
