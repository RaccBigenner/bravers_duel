# オープンβテスト 要件 v1（2026-07-23 社長決定）

## ゴール

URLを開くだけで、誰でもブラウザで **CPU対戦とデッキ構築** が遊べる。
**スマホ縦持ちがメイン**（PCでも遊べるようにする）。

## やらないこと（βの範囲外）

- ユーザーデータのサーバー保存・アカウント（**保持しない**。社長決定）
- PvP通信対戦
- カードの所持・ガチャ（全144枚を自由に使える）
- 凝ったバトル演出（βは最小限。ダメージ数字と簡易な動きまで）

## 土台

- 今の GitHub Pages（静的サイト）に同居。バトルエンジンはブラウザ内で動く（実装済み）
- カード描画コンポーネント（実装済み）を使い回す

## 画面

1. **ホーム** — 「バトル」「デッキ構築」「カード一覧（既存）」の3モード＋遊び方
2. **デッキ選択** — スタンダードデッキから選ぶ／構築デッキのJSONを読み込む
   - 自分用スタンダード: アーキタイプ8種すべてから選べる
3. **デッキ構築** — 全144枚から構築。ルール検証（キャラ枠3・50枚・同名4枚まで）つき
   - **JSON書き出し**（クリップボードコピー＆ファイル保存）
   - **JSON読み込み**（貼り付け／ファイル選択 → 検証してから使用）
   - 形式: `{ "name": "デッキ名", "characterIds": [...], "cardIds": [...] }`
4. **バトル** — CPU対戦
5. **リザルト** — 勝敗表示＋ **X(Twitter)シェアボタン**（結果文言＋βのURL）＋もう一回

## 敵（CPU）

- **スタンダード4デッキ**: 斬の勇者／闇単アグロ／氷結コントロール／聖光の癒し
  （勝率が中位の4種。強すぎる雷土の重撃と弱すぎる竜の猛攻は外す）
- **AIは1種類**: simpleAI をβ用に磨く（ガード割り込み・装備・フィールドは使用済み。
  対象選択・回復対象の質を上げる）

## バトル画面の要件（スマホ縦）

操作（プレイヤーができること全部）:

- スキルを使う（攻撃対象の選択・回復対象の選択を含む）
- キャラクターカードを使う（同名回復）
- 装備カードを付ける（対象キャラ選択）
- フィールドカードを出す
- チャージするカードを選ぶ／ターンエンド
- **ガード割り込み**（相手の攻撃時に「ガードを使う／そのまま受ける」を選ぶ）
- 投了（ホームに戻る）

表示:

- 両者のキャラクター（HP・属性・装備・アクター表示・戦闘不能）
- 自分の手札（カードデザインそのまま）、タップで拡大
- AP枚数・山札残り・トラッシュ枚数（両者）
- フィールドカード、ターン数、フェーズ表示
- 簡易ログ（何が起きたか）
